## Как думает опытный владелец Соболя

При составлении тем думай так:
- Что первым делают при покупке б/у Соболя?
- Какие регламентные работы забывают делать?
- Что чаще всего выходит из строя на пробеге 100–200 тыс. км?
- Что стоит дорого в сервисе, но можно сделать самому?
- Какие типичные болячки именно Соболя (не Газели)?
- Что важно перед зимой / перед летом?

Примеры хороших тем:
```

Антифриз — объём и марка
```

## Рабочий цикл

Повторяй для каждой темы из своего списка:

```
[Тема N/M] → поиск → синтез → сохранение → следующая тема
```

### Шаг 1 — Объяви тему

```
🎯 Тема [N/M]: [название]
   Категория: kb/[папка]/[файл].md
```

### Шаг 2 — Поиск (для каждой темы)

Ищи последовательно, пока не наберёшь 3+ источника:

```
WebSearch: "site:gazel.net [ТЕМА]"
WebSearch: "site:drive2.ru Соболь [ТЕМА]"
WebSearch: "ГАЗ Соболь [ТЕМА] форум ремонт своими руками"
WebSearch: "[компонент Соболя] [ТЕМА]"   ← если тема про конкретный узел
```

Читай треды через WebFetch — ищи практику, не рекламу.

### Шаг 3 — Два слоя

**Для INDEX.md** — 3–5 строк:
- только цифры, артикулы, ключевые факты
- главное что нельзя перепутать

**Для kb-файла** — всё остальное:
- диагностика, пошаговое описание, типичные ошибки, инструмент

### Шаг 4 — Сохрани kb-файл

```markdown
# [Тема]

> Применимость: ЗМЗ-402 / ЗМЗ-405 / ЗМЗ-406 / дизель / все
> Модели: Соболь 2217 / 2752 / 2310 / все

## Симптомы / Когда нужно
## Диагностика
## Решение
### Нюансы Соболя
### Типичные ошибки
### Инструмент и расходники
## Источники
---
*Собрано: [дата]*
```

Если файл существует — **дополняй**, не перезаписывай.

### Шаг 5 — Обнови INDEX.md

Найди секцию (`<!-- [КАТЕГОРИЯ]_START -->` … `<!-- [КАТЕГОРИЯ]_END -->`).
Добавь запись:

```markdown
### [Тема]
`[факт 1]` `[факт 2]` `[применимость]`
[одна строка — самое важное]
→ [Детали](kb/[папка]/[файл].md)
```

Обнови `*Последнее обновление: ...*`

### Шаг 6 — Кратко отчитайся и переходи к следующей теме

```
✅ [N/M] kb/[папка]/[файл].md + INDEX.md
   💡 [одна строка главного вывода]
```
